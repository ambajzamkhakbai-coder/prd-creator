# prd-creator

> 通过三步渐进式方法论引导用户从零创建完整 PRD 文档的智能工具

[English](README_en.md) | 简体中文

---

## ✨ 功能特性

- 📋 **结构化需求梳理** - 告别需求模糊、逻辑断裂的困扰
- 🔄 **分步确认机制** - 每步严格确认，确保产出质量
- 📄 **三档输出** - 需求逻辑定义 → 界面与交互设计 → PRD.md 终稿
- 🎯 **P0/P1/P2 自动优先级标记** - 启发式标注 + 判断理由，用户可逐条确认或调整
- 🗺️ **模块 Sitemap ASCII** - 按模块输出 ASCII 树状图，单模块结构一眼看清
- 🎬 **核心用户场景流程图** - 自动识别 2-4 个核心场景生成端到端 Mermaid 流程图
- 📊 **Mermaid 可视化** - 全局导航图、状态图、场景流程图让文档更直观
- 🎯 **完整结构型 PRD** - 符合专业标准的 PRD 文档格式

## 📖 工作流程

```
┌─────────────────────────────────────────────────────────────┐
│                      prd-creator                            │
│                    三步渐进式方法论                           │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│  Step 1: 讨论需求逻辑                                        │
│  ┌─────────────┐    ┌─────────────┐    ┌─────────────┐     │
│  │ 需求范围确定 │ →  │ 模块逻辑定义 │ →  │需求逻辑定义.md│     │
│  └─────────────┘    └─────────────┘    └─────────────┘     │
└─────────────────────────────────────────────────────────────┘
                              │ (用户确认)
                              ▼
┌─────────────────────────────────────────────────────────────┐
│  Step 2: 确定界面+交互设计                                    │
│  ┌─────────────────┐  ┌─────────────────┐                  │
│  │页面结构与导航流程 │→ │界面与交互设计.md│                   │
│  └─────────────────┘  └─────────────────┘                  │
└─────────────────────────────────────────────────────────────┘
                              │ (用户确认)
                              ▼
┌─────────────────────────────────────────────────────────────┐
│  Step 3: 整合输出 PRD 终稿                                    │
│  ┌─────────────┐    ┌─────────────┐    ┌─────────────┐     │
│  │需求逻辑定义  │ +  │界面与交互设计│ =  │   PRD.md   │     │
│  └─────────────┘    └─────────────┘    └─────────────┘     │
└─────────────────────────────────────────────────────────────┘
```

### Step 1: 讨论需求逻辑

1. **需求范围与优先级标记** - 确定 MVP 功能边界，自动按启发式（P0 核心闭环必需 / P1 显著体验提升 / P2 锦上添花）标注每个功能的优先级 + 判断理由，输出排序总览，用户逐条确认/调整
2. **模块逻辑定义** - 逐模块讨论需求逻辑（一次一个模块）
3. **输出** `需求逻辑定义.md`

### Step 2: 确定界面+交互设计

1. **页面结构与导航** - 梳理页面清单，绘制全局导航 Mermaid 图，按模块输出 sitemap ASCII 树状图
2. **整体界面交互** - 讨论核心页面的交互设计与单页 ASCII 布局；自动识别 2-4 个核心用户场景，生成跨模块跨页面的端到端 Mermaid 交互流程图
3. **输出** `界面与交互设计.md`

### Step 3: 整合输出 PRD 终稿

将前两步的文档整合为完整的 PRD，包含：
- 产品概述
- MVP 需求范围
- 功能需求
- 界面与交互设计
- 非功能需求
- 数据需求
- 风险与依赖

## 🔧 技术实现

- **架构模式**: 完整工作流型（模式 E）
- **自由度**: 中自由度 - 流程框架严格，内容灵活调整
- **触发机制**: 关键词 + 主动选择
- **工具支持**: AskUserQuestion, Write, Read

## 📝 创作背景

本 Skill 通过标准的 `skill-creator` 工具创建，借助 SOLO MTC 模式实现快速构建。核心设计理念：

1. **先讨论后产出** - 通过多轮对话确保需求理解准确
2. **每步确认再推进** - 避免返工，提高效率
3. **中间产物可追溯** - 需求逻辑定义 + 界面交互设计双档留存
